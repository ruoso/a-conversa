# Add `.gitignore`

**TaskJuggler entry**: [tasks/00-foundation.tji](../../00-foundation.tji) — task `foundation.repo_skeleton.gitignore`
**Effort estimate**: 0.25d
**Inherited dependencies**: `foundation.stack_decisions` (settled — TypeScript on Node)

## What this task is

Add a `.gitignore` at the repo root covering the patterns relevant to TypeScript / Node and the chosen tooling.

## Why it needs to be done

Standard hygiene. Avoids accidentally committing build artifacts, dependency caches, environment files, and editor-local state.

## Inputs / context

Patterns relevant to a TS/Node project (using pnpm workspaces if `dir_layout` lands there):

- `node_modules/`
- Build outputs: `dist/`, `build/`, `.next/`, `.svelte-kit/`, `.output/` (depending on chosen frontend stack — fold in once `frontend_framework_decision` lands)
- TypeScript: `*.tsbuildinfo`
- Coverage: `coverage/`, `.nyc_output/`
- Test artifacts: `playwright-report/`, `test-results/`
- pnpm: `.pnpm-store/` (if not centralized)
- Environment: `.env`, `.env.local`, `.env.*.local` (but **not** `.env.example`, which is checked in)
- IDE local state: `.vscode/*`, `.idea/`, `.DS_Store`, `*.swp` (or carve out `.vscode/launch.json`, `.vscode/settings.json` if shared editor config is committed)
- Logs: `*.log`, `logs/`
- TaskJuggler reports (if/when generated): `report/`, `*.html` outputs from `tj3 -o`

Plus repo-specific:

- Anything generated by the local-dev compose stack into the working directory (e.g., a `data/` volume mount for Postgres) — depends on the dev compose layout.

## Constraints / requirements

- Doesn't ignore source code or design artifacts.
- `.env.example` is checked in (template); `.env` is ignored.
- Updated once `frontend_framework_decision` lands so framework-specific build outputs are covered.

## Acceptance criteria

- `.gitignore` at root with the patterns above (where applicable for the chosen stack).
- A clean checkout-and-build produces no untracked-but-committable files.
